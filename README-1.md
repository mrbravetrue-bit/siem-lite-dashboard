# SIEM-lite — Log Analysis & Threat Detection Dashboard

A lightweight, dependency-free SIEM pipeline: parses SSH and nginx logs, runs
rule-based threat detection (brute force, endpoint scanning, request floods),
stores everything in SQLite, and visualizes it on a terminal-styled dashboard.

Built to demonstrate core SOC/blue-team concepts — log normalization,
detection engineering, and alert triage — in a small, readable codebase.

## What it does

```
raw logs  ─▶  parser.py   ─▶  normalized events  ─▶  detector.py  ─▶  alerts
(ssh/nginx)   (regex-based)   (common schema)         (rule engine)

                                    │
                                    ▼
                          storage.py (SQLite)
                                    │
                                    ▼
                          export.py → data.json
                                    │
                                    ▼
                    dashboard/ (static HTML/JS, Chart.js)
```

1. **`parser.py`** — normalizes raw `auth.log` (SSH) and nginx `access.log`
   lines into a common event schema (`timestamp`, `ip`, `event_type`, `detail`).
2. **`detector.py`** — sliding-window rule engine that flags:
   - **Brute force** — ≥5 failed SSH logins from one IP within 60s
   - **Endpoint scan** — ≥10 distinct paths hit by one IP within 30s
   - **Request flood** — ≥20 requests to the same path from one IP within 60s
3. **`storage.py`** — persists events/alerts to SQLite (`db/siem.db`).
4. **`export.py`** — aggregates everything into `dashboard/data.json`.
5. **`dashboard/`** — a static, no-backend-required dashboard (open
   `index.html` directly, or serve via GitHub Pages) that renders stats,
   charts, a top-offenders table, and a live-animated threat feed.

## Viewing the live dashboard

The dashboard is fully static — after cloning, just open `dashboard/index.html`
in your browser, or enable **GitHub Pages** (Settings → Pages → Deploy from
branch → select `main` and `/dashboard` folder) to get a shareable live link
for your portfolio/resume.

## Quick start

```bash
git clone <this-repo>
cd siem-lite-dashboard/src

python3 log_generator.py   # generates synthetic sample logs in ../data/
python3 main.py            # parses, detects, stores, exports

# then open dashboard/index.html in your browser
```

No external Python packages required — everything runs on the standard
library (`sqlite3`, `re`, `json`, `datetime`). The dashboard uses Chart.js
via CDN for charts.

## Using your own logs

Drop real `auth.log` / nginx `access.log` files into `data/` (matching the
standard syslog / combined log format) and point `main.py` at them instead
of the sample files. The parser regexes are in `src/parser.py` if your log
format differs and needs adjusting.

## Extending the detection engine

Rules live in `src/detector.py` as small, independent functions — each takes
the normalized event list and returns a list of alert dicts. To add a new
rule (e.g. geo-velocity, off-hours login, known-bad-IP lookup), write a new
`detect_*` function and register it in `run_all_detections()`.

## Why this project

This was built as a portfolio piece to demonstrate:
- Reading and normalizing heterogeneous log formats
- Writing sliding-window detection logic (a core SOC/detection-engineering skill)
- Structuring a small data pipeline (parse → detect → store → export)
- Building a clean, purpose-built UI for security data
